Definition

Linear search is a simple search algorithm used to find an element in a list or array by checking each element one-by-one in a sequential manner.

Approach

  1. Start from the first element and compare it with the target value.
  2. If the current element matches the target, return the index or the element.
  3. If no match is found after checking all elements, return a failure indication (e.g., -1).

Algorithm

  • Input: An array or list of size n and a target value.
  • Output: Index of the target if found; otherwise, -1.

Steps

  1. Initialize a loop starting from the first element (index 0).
  2. Compare each element with the target value.
  3. If the element matches the target, return the current index.
  4. If the end of the list is reached and no match is found, return -1.

Time Complexity

  • Best case: O(1) – when the target is at the first position.
  • Worst case: O(n) – when the target is at the last position or not present at all.

Space Complexity

O(1) – No extra space is used, apart from the input list and a few variables for indexing.

Usage

  • Works for both unsorted and sorted lists.
  • Common in small or unsorted datasets where a more complex search algorithm might be unnecessary.

Advantages

  • Simple to implement.
  • No need for preprocessing the data (e.g., sorting).

Disadvantages

  • Inefficient for large datasets due to its linear time complexity.
  • Not suitable for real-time applications with large data.

Example

Given a list [3, 8, 2, 7, 5] and target 7, linear search will start from index 0, check each element until it finds the target at index 3.
